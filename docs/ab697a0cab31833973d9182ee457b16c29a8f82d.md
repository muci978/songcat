# SongCat：显式曲谱选择 + 多图分组 + Webview 尺寸修复

## 上下文

SongCat 是 Electron+React+SQLite 个人曲谱练习管理器。

此前已完成搜索预览、入库后重命名、练习页两栏布局等改动（见历史提交 v0.2.11–v0.2.13）。

当前用户反馈两个新问题：

1. **不要按“主资源”自动打开**：用户在歌曲详情页想打开哪条资源就开哪条；手动批量导入多张图片/PDF 时，希望把它们关联成一组、按顺序连续浏览。
2. **预览 / 查看窗口大小仍没变化**：`GuistudyViewer` 内部 `<webview>` 高度未真正跟随容器，`calc(100vh - ...)`/`100%` 在 flex/modal 中失效，需要改成基于容器实测像素值驱动 webview 几何。

本计划在保留已有功能的基础上解决这两个问题。

---

## A) 显式选择单条资源 + 多文件分组合并浏览

### 1. DB schema 变更（migration v3）

给 `score_assets` 增加两列，用于把同一批导入的文件关联为有序组：

| Column | Type | Purpose |
|---|---|---|
|`group_id`|`TEXT`|同一批导入的共享标识；单行导入时为 `NULL` |
|`group_sort`|`INTEGER NOT NULL DEFAULT 0`|该资源在同组内的顺序索引 |

**Migration v3** (`src/main/db/migrate.ts`)：

```sql
ALTER TABLE score_assets ADD COLUMN group_id TEXT;
ALTER TABLE score_assets ADD COLUMN group_sort INTEGER NOT NULL DEFAULT 0;
CREATE INDEX IF NOT EXISTS idx_assets_song_group ON score_assets(song_id, group_id);
PRAGMA foreign_key_check;
```

同时更新 `src/main/db/schema.ts` 的基线 DDL，让新安装直接包含这两列。

### 2. 类型与映射层扩展

- `src/shared/types/models.ts`
  - `ScoreAssetRow`: 加 `group_id: string | null`、`group_sort: number`
  - `ScoreAsset`: 加 `groupId: string | null`、`groupSort: number`
- `src/main/db/mappers.ts`：`rowToScoreAsset` 中填入上述字段。
- `src/main/db/repositories/assets.ts`
  - `NewScoreAssetRecord` 扩展可选的 `groupId?: string | null`、`groupSort?: number`，并在 INSERT SQL/binding 里写入这两列。
  - `listBySong` 保持现有排序不变即可，渲染层按需过滤分组。

### 3. 服务层：批量导入时分配 group

修改 `src/shared/types/ipc.ts` 中的 `ImportFilePathInput`，加入可选字段 `groupId? : string | null` 和 `groupSort?: number`。

修改 `src/main/services/asset.ts`：

- `importFileDialog(songId)` 在选择多个文件时生成一个公共 `groupId = newId()`；只选一个则传 `undefined`/`null`。
- 循环调用 `importFilePath({ ..., groupId, groupSort: i })`，保证顺序来自系统对话框返回顺序。
- `importFilePath()` 透传这两个字段到 repository create。
- hash 去重逻辑保持不变：重复文件直接返回已存在记录，不强制改其分组。

### 4. 路由变更：支持打开指定资源

将 `src/renderer/src/router.tsx` 的练习页路由改为带可选参数：

```tsx
<Route path="/songs/:id/practice/:assetId?" element={<Practice />} />
```

使用方式：

| URL | 行为 |
|---|---|
| `/songs/{id}/practice` | 通用入口，继续沿用主资源 → guistudy → 本地 → 外链的旧 fallback |
| `/songs/{id}/practice/{assetId}` | 精确展示点击的资源；若属于多成员组则可前后翻页 |

`src/renderer/src/pages/SongDetail.tsx` 调整：

- AssetRow “查看”按钮链接改为 `/songs/${songId}/practice/${asset.id}`。
- 顶部“进入练习”按钮维持 `/songs/${songId}/practice` 作为兜底入口。
- 每行显示组内序号标签，例如 `第 N/M 页`；非组成员不显示。

### 5. Practice.tsx：按 assetId 选中 + 组内翻页

从 `useParams<{ id: string; assetId?: string }>()` 读取 `assetId`。计算 `selectedScore` 的逻辑改为：

```ts
const selectedScore =
  (assetId && detail.scores.find((s) => s.id === assetId)) ??
  primaryScore ??
  detail.scores.find((s) => s.source === 'guistudy') ??
  detail.scores.find((s) => (s.type === 'pdf' || s.type === 'image') && s.hasLocalFile) ??
  detail.scores.find((s) => s.type === 'link' && s.source !== 'guistudy')
```

新增 helper：

```ts
function siblingGroup(score: ScoreAsset, all: ScoreAsset[]): ScoreAsset[] {
  if (!score.groupId) return [score]
  return all.filter((s) => s.groupId === score.groupId).sort((a, b) => a.groupSort - b.groupSort)
}
```

当 `group.length > 1` 且不是纯 link 时在乐谱区上方渲染左右分页控件：

- “上一页”“下一页”，禁用边界状态。
- 中间显示 `N / M`。
- 点击用 `navigate(...)` 切换到相邻资源的 route param。
- Bonus：监听键盘 ←/→ 快捷键切换页面。

媒体渲染分支复用现有代码，只是把 `displayScore` 换成 `selectedScore`。

### 6. UI 细节

- 分页控件放在左栏 Card title 下方或 viewer 上方，避免遮挡内容。
- iframe/pdf/image 容器统一占满剩余空间（flex column）。

## B) Webview 预览窗口尺寸彻底修复

### 问题根因

Electron 的 `<webview>` 不像普通元素那样能可靠解析 flex 父级里的百分比高度。即使外层传了 `height="100%"` 或 `calc(88vh - 120px)`，只要任一祖先依赖 flex/min-height 决定自身高度，`<webview>` 就可能塌陷或保持默认小尺寸。此前的改动没有触及这一点——它只在 wrapper div 上用了 prop height，而真正的 `<webview>` 仍是硬编码 `calc(100vh - 160px)`。

### 解决方案

改用 `ResizeObserver` 测量容器实际像素宽高，再把具体像素写进 `<webview>` style/attrs。

`src/renderer/src/components/GuistudyViewer.tsx` 改造要点：

1. 在外部 wrapper 内再放一层相对定位的测量面（`ref={wrapRef}`），让它占据由 `height` prop 决定的全部空间。
2. `useEffect` 中用 `ResizeObserver(wrapRef.current)` 读取 `contentRect.width/height`，写入 state `{w,h}`。
3. `<webview>` 设为 absolute inset 0，并把 `width={`${w}px`}`、`height={`${h}px`}` 写成像素样式。
4. 全屏模式同样走这套测量逻辑，只是外部容器变成 fixed/inset 0/height 100vh。
5. Practice 页调用处继续传 `height="100%"`；Modal 预览可继续传固定像素高（如 `600`）或 `calc(88vh - 120px)`，因为 GuistudyViewer 会把它转成实测像素再赋给 webview。

这样无论嵌套在多深的 flex container 还是 modal body 里，webview 都会拿到明确的像素尺寸。

## 验证步骤

1. Schema
   - 启动应用，`PRAGMA user_version == 3`。
   - SQLite schema inspector 确认 `score_assets` 有 `group_id`、`group_sort`。

2. 批量导入分组
   - 在某首歌详情页一次选三张图片导入。
   - 查库：三条记录拥有相同 `group_id`，`group_sort` 依次为 0/1/2；只有首个（整歌首次）资产被标记为主资源。

3. 单曲导入
   - 单独导一张 PDF，确认 `group_id IS NULL`，`group_sort == 0`。

4. 显式选择
   - 对一首有多个本地资源的歌，分别点每条 AssetRow 的“查看”。
   - 地址栏应为 `/songs/<song>/practice/<asset-id>`，显示的正是对应资源而非主资源/fallback。

5. 通用入口兼容
   - 点顶部“进入练习”（不带 asset id），仍然先找主资源/guistudy/local/link 链，行为不回退。

6. 分组浏览
   - 进入一个三页的组图第二页，“上一页”“下一页”均可用；翻到末页后“下一页”自动 disabled；URL 同步变化；键盘方向键可用。

7. Webview 尺寸
   - guistudy 资源应填满左侧 Card；缩放窗口大小时实时跟随。
   - DevTools Elements 面板查看 `<webview>` 的 `width/height` 样式为整数像素，不再是 `%` 或失效的 `calc(100vh...)`。
   - Modal 预览弹窗也明显变大，不再塌陷。
